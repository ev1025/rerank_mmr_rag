# Rag모델 미니 프로젝트
```
<<진행 순서 가이드>>
주제 : 카드 상품 추천 대화형AI 구현 실습
1. 카드 상품 데이터 크롤링
 - 카드고릴라 사이트의 카드 탭에서 신용 및 체크카드 데이터 활용 : https://www.card-gorilla.com/card?cate=CRD
 - 추후 RAG구조 구현시 청킹을 위해 어떤 형태로 데이터를 수집해둘 것인지 판단하여 데이터 수집 진행
2. 페르소나(가상인물) 지정 3 Case 이상
 - ex) 20대 취업준비생으로 대중교통으로 매일 학원에 통원하며, 식비를 아끼기 위해 편의점에서 주로 끼니를 해결하고, 애완동물을 기르고 있음.
 - ex) 30대 여성 직장인으로 여행을 좋아해 연 2회 이상 해외여행을 다니는 중, 몸이 자주 아파 병원을 많이 이용함, 자차를 소지함
3. API를 활용한 대화형 AI 시스템 구축
 - Base 모델은 gpt-3.5-turbo로 사용
4. 모델 고도화
 - 프롬프트 엔지니어링
 - RAG 구현
5. 모델 평가
 1) RAG가 없는 gpt-3.5-turbo 모델의 응답과 RAG 구성후의 응답을 비교
 2) 페르소나 별 모델에게 추천 받은 카드를 홈페이지에서 검색하여 맞게 추천했는지 사람이 직접 검증(Human Based Evaluation)
 - 평가 횟수는 페르소나별로 최소 10번 이상 진행할 것  
※ 발표 참고사항
 - 팀별로 데이터 수집 부터 RAG 구현까지 전체 과정에 대한 코드 리뷰 형식으로 발표
 - 데이터 수집 양, 전처리 형식, 성능이 좋았던 프롬프트 및 응답 결과에 대한 내용은 발표에 필수적으로 포함시킬 것
```
